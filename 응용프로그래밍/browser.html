HTML 변천사

1. HTML 1.0

최초 HTML의 등장

전 세계에 처음으로 나타난 HTML 버전이다. 이 당시에는 웹 페이지 제작을 하는 사람들이 매우 드물었고, 웹을 이용하는 사람도 상당히 적은 시대였다. 주로 텍스트 위주의 정보 전달 목적으로 사용되었던 시기이다.

2. WWW(World Wide Web)

웹 표준을 정의하다

HTML의 사용자가 증가함에 따라 서서히 표준화를 해야 할 필요성을 느끼고, 팀 버너스리를 중심으로 각종 기관들이 모여서 웹의 표준을 정의하여 표준화를 시작했다.

3. HTML 2.0 ~ 3.2

표준화를 위한 노력

사람들이 점점 더 웹에 몰리기 시작하자 더 많은 정보를 표현하길 원하고, 현재도 사용하고 있는 실용적인 HTML 태그들이 등장하기 시작한다. 일반 사용자들에게 다양한 브라우저들이 보편화되기 시작하면서 공통적 태그들을 표현할 수 있도록 표준을 정의하는 것에 관심을 가지기 시작한 시기이다.
당시에는 브라우저들이 표준화되어있지 않아 브라우저마다 표현되는 태그들이 달랐다. 현재까지도 IE(Internet Explorer)는 표준을 따르지 않는다.

4. HTML 4.01

HTML의 안정기

안정된 HTML의 버전이다. 이 버전부터 CSS(Cascading Sytle Sheet)로 디자인적 요소를 구분하여 표현하게 되고 HTML로는 웹 페이지의 전반적 구조만을 명시하는 것을 원칙으로 한다.
하지만 W3C(WWW Consortium) 에서는 HTML이 웹 페이지 표현에 있어 한계에 봉착했다고 판단, 더 이상 HTML을 발전시키지 않을 것을 공표한다.

5. XHTML 1.0

새로운 표준의 등장

근래 등장한 XML과 HTML을 합성하여 더 확장된 태그를 사용할 수 있으며, 문법상 오류가 있다면 표현되지 않도록 검사를 엄격하게 적용하였다.
하지만 XML은 버전 호환성을 지원하지 않았기 때문에 이전 태그들로 작성된 페이지가 사용되지 못한다는 문제점이 있었고, 문법도 엄격하여 배움에 있어 어려움이 많았다.
결국 XHTML보다 사람들은 HTML을 표현할 수 있는 버전의 브라우저를 사용하면서 흥행에 대 실패한 버전이 되었다.

5. WHATWG

HTML을 끌고 가려는 움직임

웹의 뚜렷한 표준이 없다 보니 WHATWG(Web Hypertext Application Technology Working Group)가 등장하게 된다. 표준 기관이 정확하게 정의를 하지 않으니 당시 대중적인 브라우저들을 가지고 있던 기업들(Apple, Mozilla, Opera, MS)이 직접 손을 잡아 HTML을 계승하는 언어를 만들기에 이른다.
각 기업들이 뭉쳐 HTML이 충분히 사용될만한 가치가 있다고 판단하여 그들만의 방식으로 개발해서 만든 것이 현재의 HTML5 의 시초가 되었다.

7. HTML 5

완전한 표준의 정립

결국 W3C도 자세를 낮추고 XHTML을 표준에서 탈락시키게 되고, WHATWG에서 정의한 표준을 따르기로 한다. HTML5와 CSS3가 표준으로 확정되게 되었고, 이것이 현재의 HTML5가 되었다.

웹 브라우저 종류와 특징

1. Internet Explorer


마이크로소프트에서 개발, 배포하고 있었던 Windows, Windows Phone 전용 웹 브라우저. 약어는 IE이다.
이름은 말 그대로 인터넷(Internet) 탐색기(Explorer)라는 직관적인 이름을 가지고 있으며 한때 브라우저의 점유율 대부분을 차지했으나 현재는 잘 쓰이지 않는다. 2004년에 등장안 파이어폭스, 2008년 말에 등장한 구글 크롬 등과 같이 속도와 성능이 더 좋은 브라우저들이 등장하면서 점유율이 점차 떨어졌다. 한국에서는 ActiveX로 인해 강제적으로 사용되었으나 현재는 대부분 방식이 교체되어 찾아보기 힘들다. 마이크로소프트는 IE에 대한 개발을 중단하고 Microsoft Edge라는 크로미움 엔진을 기반으로 한 새로운 웹 브라우저를 주력으로 밀고 있다.

2. Chrome


구글에서 개발하는 브라우저이며, 2008년 9월 3일에 베타버전이, 12월 11일에 정식 버전이 출시되었다.
Chrome은 28버전 까지 Apple이 개발한 WebKit 엔진을 그대로 탑재하였으며 29버전부터 WebKit을 복제한 자체 개발 블링크 엔진을 사용한다. 이 원본소스인 웹킷 엔진을 사용하는 대가로, 블링크 엔진 소스코드 라이선스는 Apple이 결정하며 구글이 따른다. 이에 따라 구글은 Chrome 엔진 소스를 공개할 의무가 있어 소스코드가 공개되어있다. 브라우저가 교체됨에 따라 현재 가장 대중적으로 사용되는 브라우저가 되었다.

3. Firefox


Firefox 는 비영리 재단법인인 Mozilla 재단에서 개발된 Gecko 엔진 기반 오픈 소스 웹 브라우저이다. 초창기 코드명은 피닉스(Phoenix)였다. 이후 0.6~0.7.1 버전에서는 파이어버드(Firebird)라고 불렸다가 0.8 이후 버전부터 파이어폭스라는 이름으로 변경되었다.
파이어폭스는 WWW->모자이크->넷스케이프->모질라애플리케이션 스위트->피닉스->파이어버드->파이어폭스 로 이어지는 오랜 역사를 가진 브라우저이다.

4. Safari


Apple에서 만든 웹 브라우저. macOS 와 iOS, iPadOS를 사용한 Apple 제품의 기본 브라우저이다.
Microsoft Windows용 버전은 2012년 5월 9일 출시된 5.1.7버전 이후로 지원을 하지 않는다. KDE의 기본 브라우저인 컨커러(Konqueror)에 사용된 HTML 레이아웃 엔진인 KHTML 엔진을 기반으로 만든 오픈소스 엔진인 WebKit 엔진을 기반으로 하고 있다. macOS 류에서는 홈 어플리케이션인 만큼 웹 페이지 로드 속도가 빠르지만, Windows 와 같은 타 OS에서는 매우 낮은 성능을 보인다.

5. Opera


1995년 노르웨이의 오솔로에 설립된 오페라 소프트웨어가 개발한 웹 브라우저이다.
1996년 처음으로 공개되어 역사가 깊은 브라우저로, 2005년 모질라 파이어폭스 성공의 영향으로 구글 지원을 받아 셰어웨어에서 광거를 제거한 프리웨어로 탈바꿈했다. 오페라에서 사용되던 자체엔진 프레스토가 2013년 7월 2일에 전부 크럼에서도 쓰는 엔진인 블링크로 교체되었다. 기존에 쓰던 자체 엔진인 프레스토는 현재 오페라 미니의 서버에서 쓰이고, 안드로이드 및 PC용 브라우저에서는 블링크가 탑제된 오페라가 나오고 있다. 2016년 골든브릭실크로드라는 중국 IT업체 컨소시업에 인수되었다.
크롬 엔진을 기반으로 하는 브라우저지만 크롬에 붙어있던 각종 부가기능을 제거하고 최적화 작업을 거친 브라우저기 때문에 크롬보다 가볍고 빠르다는 특징을 가지고 있다. 프레스토 엔진을 사용하는 12이하의 클래식 버전은 펜티엄3와 같은 열악한 환경에서도 매끄러운 구동을 보여준다.

6. Whale


Naver에서 제작한 웹 브라우저.
DEVIEW 2016에서 공개된 웨일은 간편 검색, 팝업 정리, 이미지 번역 등의 기능을 내세웠다. 특히 한 탭 안에서 모든 작업을 해결할 수 있도록 검색을 간편화했다고 한다. 자체 엔진인 슬링을 개발했지만 웨일은 크로미움 기반이다. 다른 브라우저에서는 볼 수 없는 다양하고 유용한 기능들이 웨일의 가장 큰 특징이다.
